

#  Wonderland: Сеть Хоррор-Квестов

Проект представляет собой платформу для бронирования и управления хоррор-квестами. 
Разработан в рамках дипломной работы. 

Архитектура проекта — гибридная:
* **Backend:** Ruby on Rails (API, админ-панель, бизнес-логика, генерация расписания, финансы).
* **Frontend:** React + Vite + Tailwind CSS + Framer Motion (клиентская витрина).
* **База данных:** PostgreSQL.
* **Распознавание чеков/бланков (ИИ):** Tesseract OCR + MiniMagick.

---

## Как запустить проект (Docker)

Обе части проекта (Backend и Frontend) работают в изолированных контейнерах Docker.

### 1. Запуск Backend (Ruby on Rails)
Откройте терминал, перейдите в папку бэкенда и запустите сервер:
```bash
cd quest_app
rails s
## 2. Запуск контейнеров

Сборка и запуск сервисов в фоновом режиме:

\```bash
docker-compose up --build
\```

При первом запуске Docker автоматически скачает необходимые образы, установит зависимости (`bundle install` и `npm install`) и подготовит базу данных (`rails db:prepare`).

## 3. Доступ к сервисам

После успешного запуска система будет доступна по следующим адресам:

- **Интерфейс пользователя (Frontend):** [http://localhost:5173](http://localhost:5173)
- **Документация API / Сервер (Backend):** [http://localhost:3000](http://localhost:3000)
- **База данных (PostgreSQL):** `localhost:5432`

## Локальная разработка (Без Docker)

Если вы хотите запустить компоненты системы локально:

### Backend

1. Убедитесь, что в системе установлен Tesseract OCR.
2. Перейдите в папку бэкенда:
   \```bash
   cd quest_app
   \```
3. Установите зависимости:
   \```bash
   bundle install
   \```
4. Настройте подключение к БД в `config/database.yml`.
5. Накатите миграции:
   \```bash
   bin/rails db:prepare
   \```
6. Запустите сервер:
   \```bash
   bin/rails server -p 3000
   \```

### Frontend

1. Перейдите в папку фронтенда:
   \```bash
   cd frontend-quest
   \```
2. Установите зависимости:
   \```bash
   npm install
   \```
3. Запустите сервер разработки:
   \```bash
   npm run dev
   \```

## Безопасность и проверка кода

В проекте интегрированы инструменты статического анализа кода:

- **Brakeman** — поиск уязвимостей в Rails-приложении.
- **bundler-audit** — проверка используемых гемов на наличие известных уязвимостей (CVE).

Запуск проверок локально из папки `quest_app`:

\```bash
bundle exec brakeman
bundle exec bundle-audit check --update
\```
